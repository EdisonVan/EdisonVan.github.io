单元测试工具的选择

测试框架到底是一个工具，选择一个几乎不需要配置、开箱即用的框架是可以大大提高生产效率的选择!

Mocha

适合作为高度定制的测试框架！
优点：

  • 对异步的强大支持
  • 灵活，它只提供给开发者的只有一个基础测试结构
  • 其它功能性的功能如 assertions， spies，mocks，和像它们一样的其它功能需要引用添加其它库/插件来完成。

缺点：

  • 需要额外的引入众多辅助库和插件，增加了的学习成本和配置成本。
  • 一开始为 Node.js 而生的单元测试框架，对浏览器的支持并不如其对服务器做的那么好。

Jasmine

优点：

  • UI：清晰且操作简单
  • Ready-To-Go：有断言、spies、mocks，和 Sinon 能做的事差不多。和其他库的结合使用也很方便。
  • 代码覆盖检查
    • 内置了一个基于 Istanbul 的代码覆盖工具，功能强大且性能高。

Jest (推荐)

Jest 基于 Jasmine 开发而来，比 Jasmine 更加大而全，是一款由 Facebook 开源的大而全的测试框架,它的特点是开箱即用,功能强大,目前三大框架全部采用 Jest 测试。
优点：

  • 内置的断言库
  • 内置的异步支持
  • 内置的覆盖率报告
  • 强大的 TypeScript 支持
  • 性能：Jest 基于并行测试多文件，所以在大项目中的运行速度相当快
  • Globals
    • 和 Jasmine 一样，默认创建全局环境。但这一个特性确实会降低代码灵活性和健壮性，但是非常方便测试者调用。
  • 快照测试
    • Jest 快照功能由 FB 开发和维护，它还可以平移到别的框架上作为插件使用。 更强大的模块级 mocking 功能
    • Jest 允许开发者用非常简单的方法 mock 很重的库，达到提高测试效率的目的。比如可以模拟一个 promise 的 resolve，而不是真的进行网络请求。
  • 开发
    • Jest 仅仅更新被修改的文件，所以在监控模式 (watch mode) 下它的运行速度非常快
